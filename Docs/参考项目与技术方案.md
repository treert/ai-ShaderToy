# ShaderToy 桌面动态壁纸 — 参考项目与技术方案

## 项目目标

开发一个 Windows 本地 EXE 程序，运行时将桌面背景替换为实时渲染的 ShaderToy 着色器效果。

---

## 一、整体架构

```
┌──────────────────────────────────────────────┐
│  EXE 程序                                     │
│                                               │
│  1. 找到桌面壁纸窗口层 (WorkerW)                │
│  2. 将 OpenGL 渲染窗口嵌入桌面图标后面           │
│  3. 用 OpenGL 运行 ShaderToy 兼容的着色器       │
│  4. 实时渲染，替代静态壁纸                       │
└──────────────────────────────────────────────┘
```

## 二、关键技术点

### 1. 将窗口嵌入桌面背景层（WorkerW 方案）

Windows 桌面的层级结构：

```
Desktop (Progman)
  └─ SHELLDLL_DefView (图标列表)
      └─ SysListView32
```

通过向 `Progman` 发送未公开消息 `0x052C`，Windows 会创建一个 `WorkerW` 窗口，可以把渲染窗口嵌入到桌面图标的后面，实现动态壁纸效果。

```cpp
// 关键步骤伪代码
HWND progman = FindWindow("Progman", NULL);
SendMessageTimeout(progman, 0x052C, 0, 0, SMTO_NORMAL, 1000, nullptr);

// 遍历找到 WorkerW 窗口
HWND workerW = nullptr;
EnumWindows([](HWND hwnd, LPARAM lParam) -> BOOL {
    HWND shelldll = FindWindowEx(hwnd, NULL, "SHELLDLL_DefView", NULL);
    if (shelldll) {
        workerW = FindWindowEx(NULL, hwnd, "WorkerW", NULL);
    }
    return TRUE;
}, 0);

// 将渲染窗口设为 WorkerW 的子窗口
SetParent(myRenderWindow, workerW);
```

### 2. OpenGL 渲染 ShaderToy 着色器

ShaderToy 的着色器本质上是一个全屏四边形的 Fragment Shader。需要：

- 创建 OpenGL 上下文
- 绘制一个全屏四边形（两个三角形）
- 将 ShaderToy 的 `mainImage` 包装成标准的 GLSL Fragment Shader
- 提供 ShaderToy 的 uniform 变量

```glsl
#version 330 core
uniform vec3  iResolution;
uniform float iTime;
uniform float iTimeDelta;
uniform int   iFrame;
uniform vec4  iMouse;

out vec4 fragColor;

// ---- 粘贴 ShaderToy 代码 ----
void mainImage(out vec4 fragColor, in vec2 fragCoord) {
    // ...shader code...
}
// ---- ShaderToy 代码结束 ----

void main() {
    mainImage(fragColor, gl_FragCoord.xy);
}
```

### 3. ShaderToy 内置变量（Uniforms）

| 变量名 | 类型 | 说明 |
|--------|------|------|
| `iResolution` | `vec3` | 视口分辨率（像素） |
| `iTime` | `float` | 播放时间（秒） |
| `iTimeDelta` | `float` | 帧间隔时间 |
| `iFrame` | `int` | 当前帧号 |
| `iMouse` | `vec4` | 鼠标位置信息 |
| `iChannel0~3` | `sampler2D` | 输入纹理通道 |
| `iChannelResolution` | `vec3[4]` | 各通道分辨率 |
| `iDate` | `vec4` | 年/月/日/秒 |
| `iSampleRate` | `float` | 音频采样率 |

### 4. 多 Pass 渲染

ShaderToy 支持多个 Buffer（Buffer A/B/C/D）和一个 Image 输出，可实现：

- 物理模拟
- 多步后处理
- 状态存储（如流体模拟）

## 三、技术栈选择

| 方案 | 语言 | 优缺点 |
|------|------|--------|
| C++ + GLFW/GLAD + Win32 | C++ | 性能最好，体积小，但开发复杂 |
| C++ + SDL2 + OpenGL | C++ | 跨平台友好，社区资源多 |
| C + MinGW + 原生 Win32 + OpenGL | C | 极致轻量 |
| Rust + winit + glow | Rust | 安全性好，但学习曲线陡 |

**推荐方案：C++ + SDL2 + OpenGL + Win32 API**（参考 `lihp1603/ShaderToy` 的技术栈）

## 四、验证过的参考项目

### 1. rocksdanister/lively — ⭐ 18.5k stars

- **地址**：<https://github.com/rocksdanister/lively>
- **语言**：C#，WinUI 3
- **功能**：免费开源的 Windows 动态壁纸软件，支持视频、GIF、网页、ShaderToy 着色器、Unity/Godot 游戏作为桌面壁纸
- **参考价值**：完整展示了 WorkerW 桌面嵌入方案、多显示器支持、全屏应用自动暂停等，是最成熟的开源动态壁纸项目
- **许可证**：GPL-3.0

### 2. GabeRundlett/desktop-shadertoy — ShaderToy 桌面客户端

- **地址**：<https://github.com/GabeRundlett/desktop-shadertoy>
- **语言**：C++ (86.5%) + CMake + GLSL
- **功能**：ShaderToy 的全功能桌面客户端，支持从 ShaderToy 加载 `.json` 着色器、多 Pass 渲染（Buffer A/B/C/D）、所有内置 uniform
- **参考价值**：直接展示了如何在本地用 C++ + OpenGL 运行 ShaderToy 着色器，代码质量高
- **许可证**：Apache-2.0
- **平台**：Windows + Linux

### 3. lihp1603/ShaderToy — C++ + SDL + OpenGL

- **地址**：<https://github.com/lihp1603/ShaderToy>
- **语言**：C++ (99.2%)
- **功能**：用 C++ 重写的 ShaderToy 桌面版，离线环境下编写和查看着色器效果
- **技术栈**：SDL2 + GLEW + stb_image
- **参考价值**：代码结构简单直接，非常适合学习如何用 C++ 包装 ShaderToy 的 `mainImage` 函数

### 4. ShaderToy 官方 How To

- **地址**：<https://www.shadertoy.com/howto>
- **内容**：官方文档，说明了如何将 ShaderToy 着色器移植到本地 OpenGL 2/3/4 环境运行，包含 uniform 变量映射、着色器包装方式等关键信息
