# 技术架构与实现

## 技术栈
| 项目 | 选型 |
|------|------|
| 语言 | C++17 |
| 构建 | CMake 3.16+，FetchContent 自动拉取依赖 |
| 窗口/事件 | SDL2 (release-2.30.12, 静态链接) |
| OpenGL 加载 | GLAD (v0.1.36, OpenGL 3.3 Core) |
| 图片加载 | stb_image (header-only) |
| 桌面壁纸嵌入 | Win32 API (WorkerW 方案) |
| 编译器 | MSVC (Visual Studio 2022) |

## 源码结构
```
src/
├── main.cpp                # 程序入口、事件循环、命令行解析、DPI 感知
├── renderer.h/.cpp         # OpenGL 渲染器（全屏四边形 + uniform 传递）
├── shader_manager.h/.cpp   # ShaderToy 着色器加载、包装、编译（动态 sampler 类型）
├── texture_manager.h/.cpp  # iChannel 纹理管理（2D + CubeMap）
├── multi_pass.h/.cpp       # 多 Pass 渲染（Buffer A/B/C/D + Image，FBO 双缓冲）
├── file_watcher.h/.cpp     # 文件监控（轮询方式，shader 热加载）
├── tray_icon.h/.cpp        # Win32 系统托盘图标
├── wallpaper.h/.cpp        # Win32 桌面壁纸嵌入（WorkerW，多显示器）
└── stb_image.h             # stb_image 头文件

assets/shaders/
├── default.glsl            # 默认着色器（彩色渐变 + 点击涟漪）
├── texture_demo.glsl       # iChannel 纹理示例
└── my_noise_lab.glsl       # 噪声实验 shader

build-exe.bat               # 一键编译脚本
run-exe.bat                 # 快速运行脚本
```

## 命令行参数
| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--shader <path>` | ShaderToy GLSL 文件路径 | `assets/shaders/default.glsl` |
| `--channel0~3 <path>` | iChannel 纹理路径 | 无 |
| `--channeltype0~3 <t>` | 通道类型：`2d`、`cube`、`3d` | `2d` |
| `--wallpaper` | 桌面壁纸模式 | 关闭 |
| `--no-hotreload` | 禁用热加载 | 启用 |
| `--width <n>` / `--height <n>` | 窗口尺寸 | 800x600 |
| `--fps <n>` | 目标帧率 | 壁纸模式 30，窗口模式 60 |
| `--renderscale <f>` | 渲染分辨率缩放（0.1~1.0） | 壁纸模式 0.5，窗口模式 1.0 |
| `--help, -h` | 显示帮助 | — |

| 变量 | 类型 | 状态 | 说明 |
|------|------|------|------|
| `iResolution` | `vec3` | ✅ | 视口分辨率 (px) |
| `iTime` | `float` | ✅ | 播放时间 (秒) |
| `iTimeDelta` | `float` | ✅ | 帧间隔时间 (秒) |
| `iFrame` | `int` | ✅ | 帧计数 |
| `iFrameRate` | `float` | ✅ | 当前帧率 |
| `iMouse` | `vec4` | ✅ | xy=当前位置, zw=按下位置(松开取负) |
| `iDate` | `vec4` | ✅ | 年/月(0-11)/日/当天秒数 |
| `iSampleRate` | `float` | ✅ | 音频采样率（固定 44100） |
| `iChannelTime[4]` | `float` | ✅ | 各通道播放时间 |
| `iChannelResolution` | `vec3[4]` | ✅ | 各通道分辨率 |
| `iChannel0~3` | `sampler2D` / `samplerCube` | ✅ | 动态类型，取决于加载的资源 |

## iChannel 采样器类型
ShaderToy 中 iChannel 的 GLSL 类型取决于绑定的资源。`ShaderManager` 通过 `SetChannelTypes()` 在编译前配置各通道类型，`WrapShaderToySource()` 动态生成对应的 uniform 声明：
- `ChannelType::Texture2D` → `uniform sampler2D iChannelN;`
- `ChannelType::CubeMap` → `uniform samplerCube iChannelN;`
- `ChannelType::Texture3D` → `uniform sampler3D iChannelN;`

## CubeMap 加载
`TextureManager::LoadCubeMap()` 支持两种模式：
1. **6 张面文件** — 路径含 `%s`，替换为 `px/nx/py/ny/pz/nz`
2. **单张图片切割** — 自动检测布局：
   - 横条 6:1 → 依次 +X -X +Y -Y +Z -Z
   - 竖条 1:6 → 同上竖向
   - 十字 4:3 或 3:4 → 标准十字布局

## 渲染流程
```
用户 shader (mainImage)
    → ShaderManager 包装（动态 sampler 类型 + 全部 uniform 声明）
    → OpenGL 编译链接
    → Renderer 每帧：绑定纹理(2D/Cube) → 设置 uniform → 绘制全屏四边形
    → SDL swap buffer
```

## 壁纸模式特性
- **多显示器独立渲染**：每个显示器创建独立的 SDL 窗口，各自嵌入 WorkerW，用 `EnumDisplayMonitors` 获取精确的物理像素位置和分辨率
- **单 GL 上下文**：只在第一个窗口创建 GL 上下文，渲染时通过 `SDL_GL_MakeCurrent` 切换目标窗口，Shader/纹理/VAO 等资源只有一份，无需跨上下文共享
- **DPI 感知**：`SetProcessDpiAwarenessContext(PER_MONITOR_AWARE_V2)` 确保不同 DPI 的显示器获取正确的物理像素尺寸
- **鼠标局部坐标**：全局采集屏幕坐标，渲染时转换为各窗口局部坐标（含 ShaderToy Y 翻转），只有鼠标在当前显示器范围内才传入有效坐标
- **全屏自动暂停**：每秒检测前台窗口，全屏应用运行时暂停渲染
- **帧率自适应**：每 60 帧统计平均帧时间，GPU 负载高时自动降帧（最低 15fps）

## 系统托盘
壁纸模式下自动创建，右键菜单：Pause / Resume / Reload Shader / Quit

## Shader 热加载
`FileWatcher` 每 500ms 轮询文件修改时间，变化后主线程重编译。失败保留旧 shader。F5 手动触发。
