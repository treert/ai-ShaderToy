# 技术架构与实现

## 技术栈
| 项目 | 选型 |
|------|------|
| 语言 | C++17 |
| 构建 | CMake 3.16+，FetchContent 自动拉取依赖 |
| 窗口/事件 | SDL2 (release-2.30.12, 静态链接) |
| OpenGL 加载 | GLAD (v0.1.36, OpenGL 3.3 Core) |
| 图片加载 | stb_image (header-only) |
| 桌面壁纸嵌入 | Win32 API (WorkerW 方案) |
| 编译器 | MSVC (Visual Studio 2022) |

## 源码结构
```
src/
├── main.cpp                # 程序入口、事件循环、命令行解析
├── renderer.h/.cpp         # OpenGL 渲染器（全屏四边形 + uniform 传递）
├── shader_manager.h/.cpp   # ShaderToy 着色器加载、包装、编译（支持 move）
├── texture_manager.h/.cpp  # iChannel 纹理管理（stb_image 加载图片）
├── multi_pass.h/.cpp       # 多 Pass 渲染（Buffer A/B/C/D + Image，FBO 双缓冲）
├── file_watcher.h/.cpp     # 文件监控（轮询方式，shader 热加载）
├── tray_icon.h/.cpp        # Win32 系统托盘图标
├── wallpaper.h/.cpp        # Win32 桌面壁纸嵌入（WorkerW）
└── stb_image.h             # stb_image 头文件

assets/
└── shaders/
    ├── default.glsl         # 默认着色器（彩色渐变动画）
    └── texture_demo.glsl    # iChannel 纹理示例
```

## 构建与运行
```bash
# 配置（首次会自动下载 SDL2 和 GLAD）
cmake -B build

# 编译
cmake --build build --config Release

# 窗口模式
build\Release\ShaderToyDesktop.exe

# 壁纸模式
build\Release\ShaderToyDesktop.exe --wallpaper

# 自定义 shader + 纹理
build\Release\ShaderToyDesktop.exe --shader assets\shaders\texture_demo.glsl --channel0 path\to\image.png
```

## 命令行参数
| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--shader <path>` | ShaderToy GLSL 文件路径 | `assets/shaders/default.glsl` |
| `--channel0~3 <path>` | iChannel0~3 纹理图片路径 | 无 |
| `--wallpaper` | 启用桌面壁纸模式 | 关闭 |
| `--no-hotreload` | 禁用 shader 热加载 | 启用 |
| `--width <n>` | 窗口宽度（壁纸模式下自动全屏） | 800 |
| `--height <n>` | 窗口高度（壁纸模式下自动全屏） | 600 |
| `--fps <n>` | 目标帧率 | 60 |
| `--help` | 显示帮助信息 | — |

## 渲染流程
```
用户 shader (mainImage)
    → ShaderManager 包装为完整 Fragment Shader
        （自动注入 iResolution/iTime/iMouse/iChannel0~3 等 uniform 声明）
    → OpenGL 编译链接
    → Renderer 每帧：
        1. 绑定 iChannel 纹理
        2. 设置所有 uniform
        3. 绘制全屏四边形
    → SDL swap buffer 显示
```

## 多 Pass 渲染
```
Buffer A (FBO) → outputTexture A
Buffer B (FBO) → outputTexture B（可读取 Buffer A 输出）
Buffer C (FBO) → outputTexture C
Buffer D (FBO) → outputTexture D
Image pass     → 屏幕（可读取任意 Buffer 输出）

每个 buffer 使用双缓冲：当前帧写入 outputTexture，读取 outputTexturePrev（上一帧）。
帧结束后 swap，避免读写冲突。
```

## ShaderToy Uniform 支持状态
| 变量 | 类型 | 状态 | 说明 |
|------|------|------|------|
| `iResolution` | `vec3` | ✅ 已接入 | 视口分辨率 (px) |
| `iTime` | `float` | ✅ 已接入 | 播放时间 (秒) |
| `iTimeDelta` | `float` | ✅ 已接入 | 帧间隔时间 (秒) |
| `iFrame` | `int` | ✅ 已接入 | 帧计数 |
| `iMouse` | `vec4` | ✅ 已接入 | xy=当前位置, zw=按下位置(松开取负) |
| `iDate` | `vec4` | ✅ 已接入 | 年/月(0-11)/日/当天秒数 |
| `iChannel0~3` | `sampler2D` | ✅ 已接入 | 输入纹理通道（图片或 Buffer 输出） |
| `iChannelResolution` | `vec3[4]` | ✅ 已接入 | 各通道分辨率 |
| `iSampleRate` | `float` | ⬜ 已声明 | 音频采样率（待接入） |

## Shader 热加载
- 默认启用，通过 `FileWatcher` 每 500ms 轮询 shader 文件修改时间
- 检测到变化后在主线程重新编译，编译失败保留旧 shader
- 也可按 F5 手动触发
- `--no-hotreload` 禁用

## 系统托盘
壁纸模式下自动创建系统托盘图标，右键菜单：
- Pause — 暂停渲染
- Resume — 恢复渲染
- Reload Shader — 手动重载
- Quit — 退出程序

## 壁纸嵌入原理
通过 Win32 API 向 `Progman` 发送未公开消息 `0x052C`，Windows 会创建 `WorkerW` 窗口层。
将渲染窗口 `SetParent` 到该层，即可显示在桌面图标之下，实现动态壁纸效果。
