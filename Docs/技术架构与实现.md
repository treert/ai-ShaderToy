## 技术栈
| 项目 | 选型 |
|------|------|

## 依赖管理

项目的 4 个第三方依赖通过 **Git Submodule** 管理，存放在 `third_party/` 目录：

| 库 | 路径 | 版本 (Git Tag) | 用途 |
|----|------|----------------|------|
| SDL2 | `third_party/SDL2` | `release-2.30.12` | 窗口/输入/平台层 |
| GLAD | `third_party/glad` | `v0.1.36` | OpenGL 3.3 Core 加载器 |
| Dear ImGui | `third_party/imgui` | `v1.91.8-docking` | 调试 UI |
| nlohmann/json | `third_party/json` | `v3.11.3` | JSON 解析 |

### 双保险机制
CMakeLists.txt 采用**本地优先 + FetchContent 回退**策略：
1. 如果 `third_party/<lib>` 存在（已初始化 submodule），直接 `add_subdirectory()` 使用本地源码
2. 如果不存在，自动通过 `FetchContent` 从 GitHub 下载对应版本

### 首次使用
```bash
git clone --recursive <repo-url>    # clone 时自动拉取 submodule
# 或
git clone <repo-url>
git submodule update --init          # 手动初始化
```

### 更新依赖版本
```bash
cd third_party/<lib>
git fetch --tags
git checkout <new-tag>
cd ../..
git add third_party/<lib>
git commit -m "chore: update <lib> to <new-tag>"
```

## iChannel 采样器类型
ShaderToy 中 iChannel 的 GLSL 类型取决于绑定的资源。`ShaderManager` 通过 `SetChannelTypes()` 在编译前配置各通道类型，`WrapShaderToySource()` 动态生成对应的 uniform 声明：
- `ChannelType::Texture2D` → `uniform sampler2D iChannelN;`
- `ChannelType::CubeMap` → `uniform samplerCube iChannelN;`
- `ChannelType::Texture3D` → `uniform sampler3D iChannelN;`

## CubeMap 加载
`TextureManager::LoadCubeMap()` 支持两种模式：
1. **6 张面文件** — 路径含 `%s`，替换为 `px/nx/py/ny/pz/nz`
2. **单张图片切割** — 自动检测布局：
   - 横条 6:1 → 依次 +X -X +Y -Y +Z -Z
   - 竖条 1:6 → 同上竖向
   - 十字 4:3 或 3:4 → 标准十字布局

## 渲染流程
```
用户 shader (mainImage)
    → ShaderManager 包装（动态 sampler 类型 + 全部 uniform 声明）
    → OpenGL 编译链接 → 自动缓存所有 uniform location
    → 每帧渲染：
        [壁纸模式] Buffer pass 只渲染一次（所有显示器共享）
                   → 遍历显示器，跳过被遮挡的 → Image pass + blit
        [窗口模式] RenderAllPasses（完整流程）
        → [renderScale < 1.0] 渲染到低分辨率 FBO → blit 全屏四边形放大到屏幕
        → [renderScale = 1.0] 直接渲染到屏幕
        → 测量 renderTime（shader 渲染耗时）
        → fillDebugState() 填充调试数据
        → DebugUI 渲染（ImGui overlay）
        → SDL_GL_SwapWindow
        → SDL_Delay 帧率控制
```

## 壁纸模式特性
- **多显示器独立渲染**：每个显示器创建独立的 SDL 窗口，各自嵌入 WorkerW，用 `EnumDisplayMonitors` 获取精确的物理像素位置和分辨率
- **指定显示器**：`--monitor <n>` 参数可指定壁纸只在第 n 个显示器上生效（索引从 0 开始），默认所有显示器。索引超出范围时自动 fallback 到所有显示器
- **Buffer pass 共享**：多显示器场景下 Buffer pass 和 CubeMap pass 只渲染一次，Image pass 对每个显示器各渲染一次（2 个显示器时节省约 30~40% 渲染量）
- **单 GL 上下文**：只在第一个窗口创建 GL 上下文，渲染时通过 `SDL_GL_MakeCurrent` 切换目标窗口，Shader/纹理/VAO 等资源只有一份，无需跨上下文共享
- **DPI 感知**：`SetProcessDpiAwarenessContext(PER_MONITOR_AWARE_V2)` 确保不同 DPI 的显示器获取正确的物理像素尺寸
- **鼠标局部坐标**：全局采集屏幕坐标，渲染时转换为各窗口局部坐标（含 ShaderToy Y 翻转），只有鼠标在当前显示器范围内才传入有效坐标
- **全屏自动暂停**：每秒检测前台窗口，全屏应用运行时暂停整帧渲染（`--no-pause-on-fullscreen` 可禁用）
- **桌面遮挡检测**：每秒枚举顶层可见窗口，检测是否有单个窗口覆盖某显示器 ≥90% 面积（如最大化窗口），满足时该显示器跳过 Image pass 渲染；所有显示器都被遮挡时跳过整帧（含 Buffer pass）。过滤机制：DWM cloaked 隐形窗口（UWP）、系统辅助窗口类（CoreWindow/InputSink 等）、无主工具窗口。与全屏暂停共用 `--no-pause-on-fullscreen` 开关
- **帧率自适应**：每 60 帧统计平均帧时间，GPU 负载高时自动降帧（最低 1fps）
- **降分辨率渲染**：`--renderscale` < 1.0 时，先渲染到低分辨率 FBO（GL_LINEAR 双线性插值放大）。壁纸模式默认 0.5x，像素着色量减少 75%，显著降低 GPU 负载。renderScale 调回 1.0 时自动释放 BlitRenderer FBO 回收显存
- **VSync 策略**：壁纸模式关闭 VSync——WorkerW 嵌入窗口的 VSync 行为因驱动而异（可能不生效或异常），且壁纸在图标下方无需防撕裂，改用 `SDL_Delay` 手动控帧更稳定省电
- **壁纸模式默认省电**：未指定时默认 30fps + 0.5x 渲染分辨率（窗口模式默认 60fps + 1.0x），可通过 `--fps` 和 `--renderscale` 自行调节

## GPU 性能优化
| 优化项 | 描述 | 预计收益 |
|--------|------|----------|
| FBO 格式 RGBA16F | Buffer/CubeMap FBO 从 GL_RGBA32F 降为 GL_RGBA16F，与 ShaderToy 官网一致 | 显存带宽减半 |
| Buffer pass 共享 | 壁纸模式下 Buffer pass 只渲染一次，所有显示器共享结果 | N 显示器减少 (N-1)/N 的 Buffer 渲染量 |
| 桌面遮挡检测 | 按显示器粒度检测遮挡，被遮挡的显示器跳过渲染 | 桌面被挡住时 GPU 占用接近 0 |
| Uniform location 缓存 | `UniformLocations` 结构体在编译后一次性缓存，避免每帧字符串查找 | 消除每帧 60~90 次 glGetUniformLocation |
| BlitRenderer 缓存 | uTex uniform location Init 时缓存 | 微优化 |

### Vulkan 迁移评估（结论：暂不迁移）
- ShaderToy shader 本质是 fragment shader 密集型，GPU 消耗大头在每像素数学计算，非 API 调用开销
- Vulkan 核心优势（减少 draw call 开销、多线程命令录制）在「每帧只画 1~5 个全屏四边形」的场景下几乎无收益
- WorkerW 嵌入 + Vulkan swapchain 兼容性未经验证，风险较高
- GLSL 330 → SPIR-V 需要处理 uniform → UBO 布局差异，工作量巨大

## 系统托盘
壁纸模式下自动创建，右键菜单：Pause / Resume / Reload Shader / Quit

## Shader 热加载
`FileWatcher` 每 500ms 轮询文件修改时间，变化后主线程重编译。失败保留旧 shader。F5 手动触发。

## 调试 UI（Dear ImGui）

### 架构
- **DebugUIState** 结构体：main.cpp 与 DebugUI 之间的数据桥梁，包含只读展示数据（fps、renderTime、uniform 值等）、可写控制数据（paused、targetFPS、renderScale）和一次性动作标志（requestReload、requestSwitchShader 等）
- **DebugUI** 类：封装 ImGui 生命周期（Init/Shutdown/ProcessEvent/BeginFrame/Render/RenderOverlay），内部通过 `UpdateSmoothing()` 统一 EMA 平滑所有显示指标
- **fillDebugState** lambda（main.cpp）：统一填充 debugState 公共字段，窗口模式和壁纸模式共用，消除重复代码

### 窗口模式
- Tab 键切换调试面板显隐
- 信息展示：FPS、Frame Time、RenderTime（实际渲染耗时）、Shader 路径/状态、iResolution/iTime/iFrame/iMouse
- 控制区：Reload Shader、Pause/Resume、Reset Time、Target FPS 滑条、Render Scale 滑条
- Shader 选择器：扫描 assets/shaders/*.glsl，点击切换
- ImGui 事件吞噬：WantCaptureMouse/WantCaptureKeyboard 防止 UI 操作影响 iMouse

### 壁纸模式
- `--debug` 参数启用只读 Debug 叠加（RenderOverlay），固定右上角半透明窗口，不可交互
- 未加 `--debug` 时完全跳过 ImGui 初始化和渲染，零开销

### 数值平滑
所有显示指标（FPS、Frame Time、RenderTime、TimeDelta）使用 EMA（指数移动平均，系数 0.05）平滑，避免数字闪烁。通过 `DebugUI::UpdateSmoothing()` 统一处理，Render() 和 RenderOverlay() 共用。
