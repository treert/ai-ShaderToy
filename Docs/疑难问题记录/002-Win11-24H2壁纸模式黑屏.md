# 002 — Win11 24H2 Wallpaper Mode Black Screen / Win11 24H2 壁纸模式黑屏

> **Date**: 2026-03-28
> **Status**: Resolved
> **Tags**: `D3D11`, `OpenGL`, `Windows`, `DWM`, `WS_EX_NOREDIRECTIONBITMAP`

## Problem Description / 问题描述

**Symptom**: On Windows 11 24H2 (Build 26200+), the wallpaper mode renders a completely black screen. The shader is compiling and running correctly (verified via debug output), but nothing is visible on the desktop.

**Reproduction Steps**:
1. Update to Windows 11 24H2 (Build 26200+)
2. Launch wallpaper mode: `ShaderToyDesktop.exe --wallpaper --shader assets/shaders/my_noise_lab.glsl`
3. Observe: desktop shows black instead of the shader output

**Expected Behavior**: Shader renders visibly on the desktop as a dynamic wallpaper.

**Actual Behavior**: Desktop shows solid black. OpenGL rendering executes normally on the GPU (verified via RenderDoc/debug output), but the output is not composited by DWM.

**Note**: This issue does NOT affect window mode — only wallpaper mode (WorkerW-embedded windows).

## Environment / 环境信息

- **OS**: Windows 11 24H2 (Build 26200+) — **specific to this version and later**
- **GPU**: Any (issue is OS-level, not GPU-specific)
- **Rendering**: OpenGL 3.3 via SDL2 + GLAD
- **Window Embedding**: SDL window embedded as child of WorkerW via `SetParent()`
- **Previous Working Versions**: Windows 10 all versions, Windows 11 up to 23H2

## Investigation / 排查过程

### Attempt 1: Adjust WorkerW window hierarchy
- **What was tried**: Various combinations of `SetWindowLong` style flags, `SetWindowPos` z-order changes, and `ShowWindow` calls on WorkerW and its parent.
- **Result**: ❌ Did not fix
- **Observation**: The problem is at the rendering/composition layer, not the window management layer.

### Attempt 2: Research the issue online
- **What was tried**: Searched for similar issues. Found [Lively Wallpaper issue #2074](https://github.com/rocksdanister/lively/issues/2074) with a Microsoft official response explaining the architectural change.
- **Result**: ✅ Identified root cause
- **Observation**: Microsoft confirmed that Win11 24H2 changed the desktop background rendering architecture. The `Progman` window now has `WS_EX_NOREDIRECTIONBITMAP`, which breaks OpenGL rendering.

### Attempt 3: Evaluate alternative rendering backends
- **What was tried**: Evaluated four options:
  1. `WS_EX_LAYERED` + GDI Blt — works but terrible performance (CPU readback)
  2. Vulkan — high risk, WorkerW + Vulkan swapchain compatibility unverified
  3. **D3D11** — DXGI SwapChain works with `WS_EX_NOREDIRECTIONBITMAP`
  4. Keep OpenGL with workarounds — no viable workaround found
- **Result**: ✅ D3D11 chosen as the solution
- **Observation**: Reference projects (Wallpaper Engine, Lively) all use DirectX, confirming D3D11 is the correct approach.

## Root Cause / 根因分析

### Windows 11 24H2 Desktop Architecture Change

Microsoft changed the desktop background rendering architecture in Win11 24H2 to support HDR backgrounds and other new features:

1. **Before 24H2**: `Progman` window is a normal window. DWM allocates a **redirection surface** for it. OpenGL swap chain renders to this surface, and DWM composites it onto the desktop.

2. **After 24H2**: `Progman` window is created with **`WS_EX_NOREDIRECTIONBITMAP`**. This means:
   - DWM does **not** allocate a redirection surface for this window
   - OpenGL swap chain depends on the redirection surface to composite window content
   - OpenGL rendering executes on the GPU but the output **cannot be picked up by DWM**
   - Result: black screen

3. **WorkerW hierarchy change**: WorkerW is now a child of Progman (instead of a top-level sibling), requiring a fallback search path in `FindWorkerW()`.

### Why D3D11 works

D3D11's DXGI SwapChain uses a different composition path than OpenGL:
- DXGI Flip Model (`DXGI_SWAP_EFFECT_FLIP_DISCARD`) directly presents to DWM's composition engine
- It does not depend on the window's redirection surface
- `DXGI_ALPHA_MODE_PREMULTIPLIED` is fully compatible with the new architecture

### Why OpenGL doesn't work

OpenGL on Windows uses the **WGL** (Windows GL) interface, which relies on:
1. `wglMakeCurrent` → binds GL context to a window's DC (device context)
2. `SwapBuffers` → copies the back buffer to the window's **redirection surface**
3. DWM picks up the redirection surface during composition

When `WS_EX_NOREDIRECTIONBITMAP` is set, step 2 has nowhere to write, so the output is lost.

## Solution / 解决方案

Implemented a complete D3D11 rendering backend for wallpaper mode, while keeping OpenGL for window mode.

### Key Changes

| File | Change Description |
|------|--------------------|
| `src/d3d11_renderer.h/cpp` | D3D11 device/context/multi-SwapChain management, fullscreen triangle VS |
| `src/d3d11_shader_manager.h/cpp` | HLSL template + D3DCompile + ConstantBuffer + SRV binding |
| `src/d3d11_texture_manager.h/cpp` | stb_image → Texture2D/CubeMap SRV |
| `src/d3d11_multi_pass.h/cpp` | Multi-pass RTV + double buffering + CubeMap 6-face rendering |
| `src/d3d11_blit_renderer.h/cpp` | Downscale RTV + blit PS |
| `src/glsl_to_hlsl.h/cpp` | GLSL→HLSL translation (glslang + SPIRV-Cross pipeline, 100% pass rate) |
| `src/wallpaper.cpp` | Win11 24H2+ fallback: WorkerW as Progman child + `WS_CLIPCHILDREN` + SHELLDLL_DefView refresh |
| `src/main.cpp` | Dual rendering path: `useD3D11` flag, D3D11 wallpaper rendering loop |
| `CMakeLists.txt` | Link d3d11/dxgi/d3dcompiler |

### Architecture

```
main.cpp
  ├─ wallpaperMode = true → D3D11 path
  │     SDL window (no OPENGL flag) → HWND → DXGI SwapChain
  │     GLSL → HLSL translation → D3DCompile → Pixel Shader
  │     D3D11MultiPass → D3D11Renderer → Present
  │
  └─ wallpaperMode = false → OpenGL path (unchanged)
        SDL_WINDOW_OPENGL → GL context → GLAD
        ShaderManager → MultiPassRenderer → SwapWindow
```

### WorkerW Compatibility Fix (wallpaper.cpp)

```cpp
// Win11 24H2+: WorkerW is now a child of Progman (not top-level)
HWND shellView = FindWindowExW(progman, nullptr, L"SHELLDLL_DefView", nullptr);
if (shellView) {
    workerW = FindWindowExW(progman, nullptr, L"WorkerW", nullptr);
}

// Win11 24H2+: WorkerW needs WS_CLIPCHILDREN for DWM to show child window area
LONG workerStyle = GetWindowLong(workerW_, GWL_STYLE);
if (!(workerStyle & WS_CLIPCHILDREN)) {
    SetWindowLong(workerW_, GWL_STYLE, workerStyle | WS_CLIPCHILDREN);
}

// Win11 24H2+: Refresh SHELLDLL_DefView to clear snapshot layer
HWND shellView = FindWindowExW(parent, nullptr, L"SHELLDLL_DefView", nullptr);
if (shellView) {
    ShowWindow(shellView, SW_HIDE);
    Sleep(0);
    ShowWindow(shellView, SW_SHOWNORMAL);
}
```
