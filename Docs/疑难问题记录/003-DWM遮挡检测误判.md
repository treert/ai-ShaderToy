# 003 — DWM Occlusion Detection False Positives / DWM 遮挡检测误判

> **Date**: 2026-03-25
> **Status**: Resolved
> **Tags**: `Windows`, `DWM`, `UWP`, `Win32`

## Problem Description / 问题描述

**Symptom**: In wallpaper mode, the desktop is visually clear (no maximized windows covering it), but the occlusion detection reports certain monitors as "occluded", causing the wallpaper to skip rendering on those monitors. This results in the wallpaper appearing frozen or not updating.

**Reproduction Steps**:
1. Launch wallpaper mode with debug overlay enabled
2. Ensure no windows are maximized or covering the desktop
3. Observe debug output: some monitors are reported as "occluded: yes"
4. The wallpaper stops rendering on those monitors (or all monitors if all are "occluded")

**Expected Behavior**: Occlusion detection should only trigger when a real, visible window covers ≥90% of a monitor's area.

**Actual Behavior**: Occlusion detection triggers even when no visible windows are covering the desktop, because invisible "ghost" windows are being counted as occluding.

## Environment / 环境信息

- **OS**: Windows 10 / Windows 11 (all versions)
- **GPU**: Any (issue is OS-level)
- **Key Detail**: Windows creates many invisible or system-level windows that pass `IsWindowVisible()` checks but are not actually visible to the user

## Investigation / 排查过程

### Attempt 1: Simple area accumulation
- **What was tried**: Initial implementation accumulated the total area of all visible windows overlapping a monitor. If total coverage ≥ 90%, the monitor was considered occluded.
- **Result**: ❌ Severe false positives
- **Observation**: Multiple overlapping windows caused double-counting of covered area. A few small windows could add up to trigger the threshold even though most of the desktop was visible. This approach was abandoned early.

### Attempt 2: Single-window coverage check
- **What was tried**: Changed to check if any **single** window covers ≥ 90% of the monitor area (instead of accumulating). This avoids the overlap double-counting problem.
- **Result**: ⚠️ Partially fixed, but still had false positives
- **Observation**: Some invisible system windows have large rectangles that cover the entire screen, triggering the single-window check.

### Attempt 3: Filter DWM cloaked windows
- **What was tried**: Added `DwmGetWindowAttribute(hwnd, DWMWA_CLOAKED, ...)` check to skip "cloaked" windows.
- **Result**: ⚠️ Partially fixed
- **Observation**: UWP apps and virtual desktop windows are "cloaked" by DWM — they return `true` for `IsWindowVisible()` but are actually invisible. Filtering them removed a major source of false positives. However, some non-cloaked system windows still caused issues.

### Attempt 4: Filter system window classes
- **What was tried**: Added a whitelist of known system window class names to skip:
  - `Progman`, `WorkerW` — desktop shell windows
  - `Shell_TrayWnd`, `Shell_SecondaryTrayWnd` — taskbar windows
  - `Windows.UI.Core.CoreWindow` — UWP core windows (often full-screen invisible)
  - `ApplicationFrameInputSinkWindow` — UWP input sink
  - `EdgeUiInputTopWndClass`, `EdgeUiInputWndClass` — Edge UI input windows
  - `ForegroundStaging` — DWM staging window
  - `Shell_InputSwitchTopLevelWindow` — input method switcher
- **Result**: ⚠️ Further improved, but edge cases remained
- **Observation**: Some system windows don't have well-known class names but are still invisible helpers.

### Attempt 5: Filter ownerless tool windows + minimized windows + zero-area windows
- **What was tried**: Added additional filters:
  - `IsIconic(hwnd)` — skip minimized windows
  - `WS_EX_TOOLWINDOW` without owner — skip ownerless tool windows (typically system helpers)
  - Zero-area windows (`right <= left || bottom <= top`)
- **Result**: ✅ Fixed — no more false positives in testing
- **Observation**: The combination of all filters effectively eliminates ghost windows while correctly detecting real maximized/fullscreen windows.

## Root Cause / 根因分析

Windows creates many "invisible" windows that pass basic visibility checks but are not actually rendered on screen. These fall into several categories:

### Category 1: DWM Cloaked Windows

Windows 10/11 uses DWM "cloaking" to hide windows without destroying them. Common cases:
- **UWP apps on other virtual desktops**: `IsWindowVisible()` returns `true`, but the window is cloaked (invisible)
- **Suspended UWP apps**: Still have visible window handles but are cloaked
- **Virtual desktop switching**: Windows on inactive desktops are cloaked

`IsWindowVisible()` does NOT check DWM cloaking — it only checks the `WS_VISIBLE` style flag. The only reliable way to detect cloaked windows is `DwmGetWindowAttribute(DWMWA_CLOAKED)`.

### Category 2: System Auxiliary Windows

Windows creates full-screen or large invisible windows for various system functions:
- `Windows.UI.Core.CoreWindow` — UWP core window, often covers the entire screen
- `ApplicationFrameInputSinkWindow` — captures input for UWP apps
- `EdgeUiInputTopWndClass` — Edge swipe gesture detection
- `ForegroundStaging` — DWM uses this for window transition animations
- `Shell_InputSwitchTopLevelWindow` — input method switching overlay

These windows are not cloaked (so `DWMWA_CLOAKED` doesn't catch them) but are effectively invisible to the user.

### Category 3: Ownerless Tool Windows

Some system processes create `WS_EX_TOOLWINDOW` windows without an owner. These are typically background helper windows that don't appear in the taskbar or Alt+Tab. They may have large rectangles but are not visible.

### Why accumulation-based detection fails

The original approach of accumulating all window areas was fundamentally flawed because:
1. Overlapping windows cause double-counting (two 50% windows → 100% false positive)
2. Many small system windows can add up to exceed the threshold
3. No way to distinguish "real" coverage from "overlapping" coverage without complex computational geometry

The single-window check (any single window ≥ 90%) is much more robust because a real maximized window will always cover ≥ 90% of its monitor by itself.

## Solution / 解决方案

Multi-layered filtering in `IsMonitorOccluded()`:

### Key Changes

| File | Change Description |
|------|--------------------|
| `src/main.cpp` | `IsMonitorOccluded` lambda with comprehensive window filtering |

### Filter Pipeline

```
EnumTopLevelWindows
  │
  ├── IsWindowVisible? ─── No → skip
  │
  ├── DWM Cloaked? ─── Yes → skip (UWP/virtual desktop ghost windows)
  │
  ├── Desktop shell class? ─── Yes → skip
  │   (Progman, WorkerW, Shell_TrayWnd, Shell_SecondaryTrayWnd)
  │
  ├── System auxiliary class? ─── Yes → skip
  │   (CoreWindow, InputSink, EdgeUI, ForegroundStaging, InputSwitch)
  │
  ├── Minimized? ─── Yes → skip
  │
  ├── Ownerless tool window? ─── Yes → skip
  │   (WS_EX_TOOLWINDOW && no owner)
  │
  ├── Zero area? ─── Yes → skip
  │
  └── Single window covers ≥90% of monitor? ─── Yes → OCCLUDED
```

### Code Highlights

```cpp
auto IsMonitorOccluded = [](int monX, int monY, int monW, int monH) -> bool {
    RECT monRect = {monX, monY, monX + monW, monY + monH};
    long monArea = (long)monW * monH;
    if (monArea <= 0) return false;

    HWND hwnd = GetTopWindow(nullptr);
    while (hwnd) {
        if (!IsWindowVisible(hwnd)) { /* skip */ }

        // Filter 1: DWM cloaked windows
        DWORD cloaked = 0;
        if (SUCCEEDED(DwmGetWindowAttribute(hwnd, DWMWA_CLOAKED, &cloaked, sizeof(cloaked)))
            && cloaked != 0) { /* skip */ }

        // Filter 2: Desktop shell windows
        wchar_t className[256] = {};
        GetClassNameW(hwnd, className, 256);
        if (wcscmp(className, L"Progman") == 0 || ...) { /* skip */ }

        // Filter 3: System auxiliary windows
        if (wcscmp(className, L"Windows.UI.Core.CoreWindow") == 0 || ...) { /* skip */ }

        // Filter 4: Minimized windows
        if (IsIconic(hwnd)) { /* skip */ }

        // Filter 5: Ownerless tool windows
        LONG_PTR exStyle = GetWindowLongPtrW(hwnd, GWL_EXSTYLE);
        if ((exStyle & WS_EX_TOOLWINDOW) && GetWindow(hwnd, GW_OWNER) == nullptr) { /* skip */ }

        // Filter 6: Zero-area windows
        RECT wndRect;
        if (wndRect.right <= wndRect.left || wndRect.bottom <= wndRect.top) { /* skip */ }

        // Check: single window covers ≥90% of monitor
        RECT inter;
        if (IntersectRect(&inter, &wndRect, &monRect)) {
            long long singleCover = (inter.right - inter.left) * (inter.bottom - inter.top);
            if (singleCover >= monArea * 90 / 100) return true;
        }

        hwnd = GetNextWindow(hwnd, GW_HWNDNEXT);
    }
    return false;
};
```

### Execution frequency

The occlusion check runs **once per second** (not every frame) to minimize overhead:
```cpp
if (now_ms - fullscreenCheckTimer > 1000) {
    for (auto& ww : wallpaperWindows) {
        ww.occluded = IsMonitorOccluded(ww.x, ww.y, ww.width, ww.height);
    }
}
```
