# ShaderToy Desktop 使用说明

## 简介

ShaderToy Desktop 是一个 Windows 桌面应用，可以运行 [shadertoy.com](https://www.shadertoy.com) 兼容的 GLSL 着色器。支持作为普通窗口运行，也支持作为桌面动态壁纸。

---

## 快速开始

### 1. 编译

需要安装 CMake 3.16+ 和 Visual Studio 2022。

```bash
cmake -B build
cmake --build build --config Release
```

首次编译会自动下载 SDL2 和 GLAD 依赖，需要网络连接。

### 2. 运行

编译完成后，可执行文件位于 `build\Release\ShaderToyDesktop.exe`。

```bash
# 直接运行（使用内置默认 shader）
build\Release\ShaderToyDesktop.exe
```

窗口中会显示一个彩色渐变动画效果。按 **ESC** 退出。

---

## 运行模式

### 窗口模式（默认）

```bash
ShaderToyDesktop.exe
```

- 窗口可自由调整大小
- 按 **ESC** 退出
- 按 **F5** 手动重载 shader

### 壁纸模式

```bash
ShaderToyDesktop.exe --wallpaper
```

- 渲染画面会嵌入桌面背景，显示在桌面图标下方
- 自动全屏匹配桌面分辨率
- 系统托盘会出现程序图标，右键菜单可以：
  - **Pause** — 暂停渲染（节省 GPU）
  - **Resume** — 恢复渲染
  - **Reload Shader** — 重新加载着色器
  - **Quit** — 退出程序

## 使用自定义 Shader

### 从 ShaderToy 网站获取 shader

1. 在 [shadertoy.com](https://www.shadertoy.com) 上找到喜欢的 shader
2. 复制 shader 代码（包含 `void mainImage(...)` 函数的部分）
3. 保存为 `.glsl` 文件，例如 `my_shader.glsl`
4. 运行：

```bash
ShaderToyDesktop.exe --shader my_shader.glsl
```

### Shader 文件格式

只需包含 `mainImage` 函数即可，**不需要** `#version` 声明和 uniform 声明，程序会自动添加：

```glsl
void mainImage(out vec4 fragColor, in vec2 fragCoord)
{
    vec2 uv = fragCoord / iResolution.xy;
    vec3 col = 0.5 + 0.5 * cos(iTime + uv.xyx + vec3(0.0, 2.0, 4.0));
    fragColor = vec4(col, 1.0);
}
```

### 可用的内置变量

在 shader 中可以直接使用以下变量：

| 变量 | 类型 | 说明 |
|------|------|------|
| `iResolution` | `vec3` | 窗口分辨率（像素），z 分量固定为 1.0 |
| `iTime` | `float` | 程序启动后经过的秒数 |
| `iTimeDelta` | `float` | 上一帧到当前帧的时间间隔（秒） |
| `iFrame` | `int` | 当前帧编号（从 0 开始） |
| `iMouse` | `vec4` | 鼠标信息，xy=当前位置，zw=点击位置 |
| `iDate` | `vec4` | 当前日期，x=年 y=月(0-11) z=日 w=当天已过秒数 |
| `iChannel0~3` | `sampler2D` | 输入纹理通道 |
| `iChannelResolution` | `vec3[4]` | 各纹理通道的分辨率 |

## 使用纹理

可以将图片（PNG、JPG、BMP 等）加载为 shader 的输入纹理：

```bash
# 加载一张图片到 iChannel0
ShaderToyDesktop.exe --shader assets\shaders\texture_demo.glsl --channel0 my_image.png

# 同时加载多张图片
ShaderToyDesktop.exe --shader my_shader.glsl --channel0 tex0.png --channel1 tex1.jpg
```

在 shader 中用 `texture(iChannel0, uv)` 采样纹理。

---

## Shader 热加载

默认启用。编辑 shader 文件保存后，程序会自动检测变化并重新编译，**无需重启**。

- 如果编译成功，画面立即更新
- 如果编译失败，控制台会输出错误信息，画面保持上一次成功的 shader
- 按 **F5** 可手动触发重载
- `--no-hotreload` 参数可禁用此功能

这非常适合一边编辑 shader 一边实时预览效果。

## 全部命令行参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--shader <path>` | 指定 ShaderToy GLSL 文件 | `assets/shaders/default.glsl` |
| `--channel0 <path>` | iChannel0 纹理图片 | 无 |
| `--channel1 <path>` | iChannel1 纹理图片 | 无 |
| `--channel2 <path>` | iChannel2 纹理图片 | 无 |
| `--channel3 <path>` | iChannel3 纹理图片 | 无 |
| `--wallpaper` | 桌面壁纸模式 | 关闭 |
| `--no-hotreload` | 禁用 shader 热加载 | 启用 |
| `--width <n>` | 窗口宽度 | 800 |
| `--height <n>` | 窗口高度 | 600 |
| `--fps <n>` | 目标帧率 | 60 |
| `--help` | 显示帮助信息 | — |

## 使用示例

```bash
# 1. 最简运行
ShaderToyDesktop.exe

# 2. 指定 shader
ShaderToyDesktop.exe --shader my_cool_effect.glsl

# 3. 带纹理的 shader
ShaderToyDesktop.exe --shader texture_demo.glsl --channel0 photo.jpg

# 4. 作为桌面壁纸
ShaderToyDesktop.exe --wallpaper --shader my_shader.glsl

# 5. 自定义窗口大小和帧率
ShaderToyDesktop.exe --shader my_shader.glsl --width 1280 --height 720 --fps 30

# 6. 查看帮助
ShaderToyDesktop.exe --help
```

## 常见问题

**Q: 运行后窗口是黑屏？**
A: 检查控制台输出是否有 shader 编译错误。确保 shader 文件路径正确。

**Q: 壁纸模式没效果？**
A: 壁纸模式依赖 Windows 的 WorkerW 机制，某些桌面管理软件可能会冲突。如果嵌入失败，程序会自动降级为窗口模式。

**Q: ShaderToy 上的 shader 复制过来不能用？**
A: 目前支持单 pass（Image）的 shader。如果 shader 用到了 Buffer A/B/C/D 多 pass 或特殊的 iChannel 输入（如视频、音频、立方体贴图），暂不支持通过命令行直接配置多 pass。

**Q: 如何退出壁纸模式？**
A: 右键系统托盘图标，选择 Quit。
