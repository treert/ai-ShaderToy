# ShaderToy Desktop 使用说明

## 简介

ShaderToy Desktop 是一个 Windows 桌面应用，可以运行 [shadertoy.com](https://www.shadertoy.com) 兼容的 GLSL 着色器。支持作为普通窗口运行，也支持作为桌面动态壁纸。

---

## 快速开始

### 1. 编译

需要 CMake 3.16+ 和 Visual Studio 2022（提供 MSVC 编译器和 Ninja 构建工具）。

**推荐**：先初始化 submodule（需要 Git），这样编译时无需联网：
```bash
git submodule update --init --recursive
build-exe.bat
```

如果没有初始化 submodule，CMake 会自动通过 FetchContent 从 GitHub 下载依赖（需要网络）。

> **构建系统说明**：`build-exe.bat` 使用 Ninja 生成器 + MSVC 编译器进行构建，同时自动生成 `compile_commands.json` 供 clangd 等代码分析工具使用。

### 2. 运行

```bash
build\ShaderToyDesktop.exe
```

显示彩色渐变动画，点击有涟漪效果。按 **ESC** 退出。

---

## 运行模式

### 窗口模式（默认）

- 窗口可调整大小，按 **ESC** 退出，**F5** 重载 shader
- 按 **Tab** 键打开/关闭调试面板

#### 调试面板功能

按 **Tab** 键可切换显示一个半透明的调试面板，提供以下功能：

- **信息展示**：实时 FPS、帧时间（Frame Time）、渲染耗时（RenderTime）、shader 路径与编译状态、iResolution/iTime/iFrame/iMouse 等 uniform 值
- **编译错误**：shader 编译失败时以红色文字显示完整错误信息
- **控制按钮**：
  - Reload Shader — 重新加载当前 shader（等同于 F5）
  - Pause / Resume — 暂停/恢复渲染（暂停时画面冻结，面板仍可操作）
  - Reset Time — 将 iTime 归零，重新开始动画
  - Browse Shader... — 打开文件对话框选择 shader（支持 .glsl/.json 文件或包含 image.glsl 的目录）
- **滑条调节**：
  - Target FPS — 调整目标帧率（15~120）
  - Render Scale — 调整渲染分辨率缩放（0.1~1.0）
- **Shader 选择器**：列出 `assets/shaders/` 目录下所有 `.glsl` 文件，点击即可切换到对应 shader，当前使用的 shader 以绿色高亮显示

### 壁纸模式

```bash
ShaderToyDesktop.exe --wallpaper
```

- 嵌入桌面背景，每个显示器独立渲染（支持不同分辨率、不同 DPI）
- 鼠标点击只在对应屏幕上产生效果，坐标精确
- 全屏应用运行时自动暂停渲染
- 系统托盘右键菜单：Pause / Resume / Reload Shader / **Debug Overlay（开关调试信息）** / **Open Shader...（文件对话框选择 shader）** / **Switch Shader（子菜单列出所有可用 shader）** / Quit
- **托盘 Tooltip**：鼠标悬浮在托盘图标上显示实时状态（FPS、RenderTime、当前 shader、monitor 索引）
- **GPU 省电优化**：默认 30fps + 0.5x 降分辨率渲染，可通过 `--fps` 和 `--renderscale` 调节
- **调试信息叠加**：加 `--debug` 参数可在壁纸画面右上角叠加显示 FPS、RenderTime、shader 状态等调试信息（只读，不可交互）

## 使用自定义 Shader

从 [shadertoy.com](https://www.shadertoy.com) 复制 shader 代码，保存为 `.glsl` 文件：

```bash
ShaderToyDesktop.exe --shader my_shader.glsl
```

只需包含 `mainImage` 函数，不需要 `#version` 和 uniform 声明。

### 多 Pass Shader

支持两种方式加载多 Pass shader：

**方式一：ShaderToy JSON 导入**

从 ShaderToy 网站导出的 JSON 文件直接加载（兼容 API 格式和直接导出格式）：

```bash
ShaderToyDesktop.exe --shader my_effect.json
```

程序会自动解析 JSON 中的 `renderpass` 数组，识别 Image/Buffer/Common 各 pass，并通过 output id 映射正确绑定 Buffer 间的引用关系。

**纹理资源**：ShaderToy JSON 中引用的纹理路径（如 `/media/a/xxx.jpg` 或 `/presets/tex00.jpg`）会按以下顺序查找本地文件：

1. JSON 文件同目录下的同名文件
2. `assets/` + 原始路径（去掉前导 `/`），如 `assets/media/a/xxx.jpg`
3. `assets/` + 文件名
4. 找不到则输出 warning，不影响加载（该通道为黑色）

建议将 ShaderToy 公共纹理保存到对应的本地目录：

```
assets/
├── media/a/       -- /media/a/HASH.ext 纹理（hash 命名的图片）
└── presets/        -- /presets/texNN.ext 和 cubeNN_0.ext 纹理
```

> **注意**：ShaderToy 的 `keyboard` 输入类型暂不支持（会被静默忽略）。`sound` pass 类型暂不支持。`cubemap` (Cube A) pass 已支持。

**方式二：多文件目录模式**

将 shader 文件放在一个目录中，按约定命名：

```
my_effect/                 -- 目录名即项目名
  image.glsl               -- Image pass（必须，最终输出到屏幕的 shader）
  buf_a.glsl               -- Buffer A（可选）
  buf_b.glsl               -- Buffer B（可选）
  buf_c.glsl               -- Buffer C（可选）
  buf_d.glsl               -- Buffer D（可选）
  cube_a.glsl              -- Cube A pass（可选，使用 mainCubemap 函数，渲染到 samplerCube）
  common.glsl              -- 共享代码（可选，会注入到每个 pass 编译时）
  channels.json            -- 通道配置（可选，指定各 pass 的 iChannel 绑定）
```

| 文件 | 是否必须 | 说明 |
|------|----------|------|
| `image.glsl` | **必须** | Image pass，最终输出画面的 shader |
| `buf_a.glsl` | 可选 | Buffer A，渲染到离屏 FBO，可被其他 pass 读取 |
| `buf_b.glsl` | 可选 | Buffer B |
| `buf_c.glsl` | 可选 | Buffer C |
| `buf_d.glsl` | 可选 | Buffer D |
| `cube_a.glsl` | 可选 | Cube A pass，使用 `mainCubemap(out vec4, in vec2, in vec3, in vec3)` 函数，渲染到 `samplerCube` |
| `common.glsl` | 可选 | 共享 GLSL 代码（函数、结构体、常量等），编译时自动注入到每个 pass 的 uniform 声明之后、用户代码之前 |
| `channels.json` | 可选 | 各 pass 的 iChannel0~3 绑定配置 |

```bash
ShaderToyDesktop.exe --shader my_effect/
```

#### channels.json 格式说明

`channels.json` 用于指定每个 pass 的 `iChannel0` ~ `iChannel3` 分别绑定到什么输入。如果不提供此文件，默认行为是：Image 的 `iChannel0` 绑定 Buffer A，Buffer A 的 `iChannel0` 自引用（读取自身上一帧输出）。

**JSON 结构**：顶层 key 是 pass 名称，值是该 pass 的通道绑定对象：

```json
{
  "image": {
    "iChannel0": "buf_a",
    "iChannel1": "buf_b",
    "iChannel2": "noise.png",
    "iChannel3": "../textures/photo.jpg"
  },
  "buf_a": {
    "iChannel0": "buf_a",
    "iChannel1": "buf_b"
  },
  "buf_b": {
    "iChannel0": "buf_a"
  }
}
```

**pass 名称（顶层 key）**：

| key | 对应 pass |
|-----|-----------|
| `image` | Image pass |
| `buf_a` | Buffer A |
| `buf_b` | Buffer B |
| `buf_c` | Buffer C |
| `buf_d` | Buffer D |

**通道绑定值（iChannel0~3 的值）**：

| 值 | 含义 |
|----|------|
| `"buf_a"` 或 `"buffer_a"` 或 `"Buffer A"` | 绑定 Buffer A 的输出纹理 |
| `"buf_b"` 或 `"buffer_b"` 或 `"Buffer B"` | 绑定 Buffer B 的输出纹理 |
| `"buf_c"` 或 `"buffer_c"` 或 `"Buffer C"` | 绑定 Buffer C 的输出纹理 |
| `"buf_d"` 或 `"buffer_d"` 或 `"Buffer D"` | 绑定 Buffer D 的输出纹理 |
| 文件路径（如 `"noise.png"`） | 绑定外部纹理（相对路径基于 shader 目录解析） |

**完整示例**：一个反馈模糊效果，Buffer A 读取自身上一帧做模糊，Image 读取 Buffer A 显示结果：

```json
{
  "image": {
    "iChannel0": "buf_a"
  },
  "buf_a": {
    "iChannel0": "buf_a"
  }
}
```

**带外部纹理的示例**：Buffer A 在 iChannel1 上使用一张噪声纹理：

```json
{
  "image": {
    "iChannel0": "buf_a"
  },
  "buf_a": {
    "iChannel0": "buf_a",
    "iChannel1": "noise.png"
  }
}
```

> **提示**：Buffer 绑定自身（如 `buf_a` 的 `iChannel0` 设为 `"buf_a"`）表示读取上一帧的输出，这是实现反馈/累积效果的常用手法，与 ShaderToy 的 Buffer 自引用行为一致。

### 可用的内置变量

| 变量 | 类型 | 说明 |
|------|------|------|
| `iResolution` | `vec3` | 窗口分辨率（像素） |
| `iTime` | `float` | 程序启动后经过的秒数 |
| `iTimeDelta` | `float` | 帧间隔时间（秒） |
| `iFrame` | `int` | 当前帧编号 |
| `iFrameRate` | `float` | 当前帧率 |
| `iMouse` | `vec4` | xy=当前位置, zw=点击位置(松开取负) |
| `iDate` | `vec4` | 年/月(0-11)/日/当天秒数 |
| `iSampleRate` | `float` | 音频采样率（44100） |
| `iChannel0~3` | `sampler2D` / `samplerCube` | 输入纹理（类型取决于资源） |
| `iChannelTime[4]` | `float` | 各通道播放时间 |
| `iChannelResolution` | `vec3[4]` | 各通道分辨率 |
| `iClickTime` | `float` | 最近点击时间（自定义扩展） |

## 使用纹理

### 2D 纹理

```bash
ShaderToyDesktop.exe --shader texture_demo.glsl --channel0 photo.png
```

Shader 中用 `texture(iChannel0, uv)` 采样。

### CubeMap（立方体贴图）

**6 张面图**（路径含 `%s`，自动替换为 px/nx/py/ny/pz/nz）：

```bash
ShaderToyDesktop.exe --shader sky.glsl --channel0 skybox_%s.png --channeltype0 cube
```

**单张图片**（自动检测十字 4:3/3:4、横条 6:1、竖条 1:6 布局并切割）：

```bash
ShaderToyDesktop.exe --shader sky.glsl --channel0 skybox_cross.png --channeltype0 cube
```

Shader 中用 `texture(iChannel0, direction)` 采样（direction 为 vec3）。

## Shader 热加载

默认启用。编辑 shader 文件保存后自动重新编译，无需重启。编译失败时保留旧 shader。按 **F5** 手动触发。`--no-hotreload` 禁用。

---

## 翻译模式（GLSL → HLSL）

使用 `--translate` 参数可将 shader 从 GLSL 翻译为完整的 HLSL Pixel Shader 源码，保存到 `Logs` 目录后退出。不会初始化窗口或图形设备。

```bash
# 单 pass shader → Logs/my_shader.hlsl
ShaderToyDesktop.exe --translate --shader my_shader.glsl

# 多 pass 目录 → Logs/multipass_demo/image.hlsl, buf_a.hlsl, ...
ShaderToyDesktop.exe --translate --shader assets/shaders/multipass_demo/

# JSON 多 pass → Logs/a_black_hole/image.hlsl, buf_a~d.hlsl
ShaderToyDesktop.exe --translate --shader assets/shaders/a_black_hole.json
```

**输出规则：**

| 场景 | 输出位置 |
|------|----------|
| 单 pass（纯 `.glsl` 文件） | `Logs/<shader_name>.hlsl` |
| 多 pass（目录/JSON） | `Logs/<shader_name>/image.hlsl`、`buf_a.hlsl` ~ `buf_d.hlsl`、`cube_a.hlsl`、`common.hlsl`（按需） |

每个 pass 的 HLSL 文件都是完整的、可独立编译的 Pixel Shader，包含 cbuffer 声明、纹理/采样器声明、辅助函数、Common 段和 main 入口。

## 全部命令行参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--shader <path>` | ShaderToy GLSL 文件 | `assets/shaders/default.glsl` |
| `--channel0~3 <path>` | iChannel 纹理图片 | 无 |
| `--channeltype0~3 <t>` | 通道类型：`2d`、`cube` | 2d |
| `--wallpaper` | 桌面壁纸模式 | 关闭 |
| `--debug` | 壁纸模式显示只读调试信息叠加 | 关闭 |
| `--no-hotreload` | 禁用热加载 | 启用 |
| `--translate` | 翻译模式：将 shader 从 GLSL 翻译为 HLSL 并保存到 Logs 目录后退出 | 关闭 |
| `--width <n>` | 窗口宽度 | 800 |
| `--height <n>` | 窗口高度 | 600 |
| `--fps <n>` | 目标帧率 | 壁纸 30 / 窗口 60 |
| `--renderscale <f>` | 渲染分辨率缩放 0.1~1.0（低于 1.0 时先渲染到低分辨率 FBO 再双线性放大，降低 GPU 负载） | 壁纸 0.5 / 窗口 1.0 |
| `--help, -h` | 显示帮助 | — |

> **说明**：`--fps` 和 `--renderscale` 在两种模式下都可用，只是未指定时的**默认值不同**——壁纸模式侧重省电（30fps + 半分辨率），窗口模式侧重画质（60fps + 原生分辨率）。

## 使用示例

```bash
# 最简运行
ShaderToyDesktop.exe

# 指定 shader
ShaderToyDesktop.exe --shader my_effect.glsl

# 带 2D 纹理
ShaderToyDesktop.exe --shader texture_demo.glsl --channel0 photo.jpg

# 带 CubeMap
ShaderToyDesktop.exe --shader sky.glsl --channel0 skybox_%s.png --channeltype0 cube

# 桌面壁纸（默认 30fps + 0.5x 渲染分辨率，自动省 GPU）
ShaderToyDesktop.exe --wallpaper --shader my_shader.glsl

# 壁纸模式想要更高画质
ShaderToyDesktop.exe --wallpaper --shader my_shader.glsl --renderscale 0.75

# 壁纸模式极致省电
ShaderToyDesktop.exe --wallpaper --shader my_shader.glsl --fps 20 --renderscale 0.3

# 壁纸模式原生分辨率（和窗口模式一样的画质）
ShaderToyDesktop.exe --wallpaper --shader my_shader.glsl --fps 60 --renderscale 1.0

# 壁纸模式显示调试信息
ShaderToyDesktop.exe --wallpaper --debug --shader my_shader.glsl

# 自定义窗口和帧率
ShaderToyDesktop.exe --shader my_shader.glsl --width 1280 --height 720 --fps 30

# 多 Pass 目录模式
ShaderToyDesktop.exe --shader assets/shaders/multipass_demo/

# 多 Pass JSON 模式（ShaderToy API 导出的 JSON）
ShaderToyDesktop.exe --shader exported_shader.json

# 翻译模式：GLSL → HLSL（单 pass 输出到 Logs/<name>.hlsl）
ShaderToyDesktop.exe --translate --shader my_shader.glsl

# 翻译模式：多 pass 输出到 Logs/<name>/ 子目录
ShaderToyDesktop.exe --translate --shader assets/shaders/multipass_demo/

# 查看帮助
ShaderToyDesktop.exe -h
```

## 常见问题

**Q: 运行后窗口是黑屏？**
A: 检查控制台输出是否有 shader 编译错误，确保文件路径正确。

**Q: 壁纸模式没效果？**
A: 某些桌面管理软件可能冲突。嵌入失败时会自动降级为窗口模式。

**Q: ShaderToy 上的 shader 不能用？**
A: 支持单 pass（Image）、多 pass（Buffer A/B/C/D + Image）和 Cube A (CubeMap) pass。单 pass 直接保存为 .glsl 文件；多 pass 可通过 ShaderToy API 导出为 JSON，或手动拆分到多文件目录中。Sound pass 暂不支持。

**Q: 4K 显示器只显示了一部分？**
A: 程序已支持 Per-Monitor DPI Aware，应能正确覆盖。如仍有问题请反馈。

**Q: 如何退出壁纸模式？**
A: 右键系统托盘图标，选择 Quit。

**Q: 壁纸模式 GPU 占用太高？**
A: 壁纸模式已默认启用省电优化（30fps + 0.5x 渲染分辨率）。如仍嫌高，可用 `--fps 20 --renderscale 0.3` 进一步降低。如果画面模糊，可用 `--renderscale 0.75` 提高画质。

## 辅助工具

### json2dir.py — JSON 转目录格式

将 ShaderToy 导出的 JSON shader 文件转换为项目支持的多文件目录格式，便于编辑和调试。

**用法：**

```bash
python tools/json2dir.py <input.json> [-o <output_dir>] [--force]
```

**参数：**

| 参数 | 说明 |
|------|------|
| `<input.json>` | 输入的 ShaderToy JSON 文件路径 |
| `-o <output_dir>` | 输出目录路径（默认：与 JSON 同目录，以文件名为目录名） |
| `--force` | 输出目录已存在时强制覆盖 |

**示例：**

```bash
# 转换黑洞 shader，输出到 assets/shaders/a_black_hole/
python tools/json2dir.py assets/shaders/a_black_hole.json --force

# 指定输出目录
python tools/json2dir.py assets/shaders/a_black_hole.json -o assets/shaders/black_hole

# 转换单 pass shader
python tools/json2dir.py assets/shaders/my_sdf.json --force
```

**功能说明：**
- 兼容两种 JSON 格式：有 `"Shader"` 包裹层（API 格式）和无包裹层（直接导出格式）
- 兼容两套字段名：`type`/`filepath`/`id` 和 `ctype`/`src`
- 自动识别 Image / Buffer A~D / Common 三种 pass 类型
- 通过 output id 映射表精确解析 Buffer 间引用关系
- 生成 `channels.json` 通道绑定配置（buffer 引用 + 外部纹理路径）
- keyboard 输入类型静默忽略
- 输出转换摘要（pass 数量、通道绑定详情）
