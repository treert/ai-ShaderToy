# D3D11 壁纸渲染后端

## 1. 背景

### 1.1 问题

Windows 11 24H2 (Build 26200+) 更改了桌面壁纸窗口的底层架构，导致 OpenGL 渲染的动态壁纸**黑屏**。

微软官方技术说明（来自 [Lively issue #2074](https://github.com/rocksdanister/lively/issues/2074)）：

> 为了支持 HDR 背景等新场景，Windows 改变了背景渲染方式：
> - 当桌面从列表视图窗口分离（"raised desktop"）时，不再创建多个顶级 HWND
> - 顶级 `Progman` 窗口现在创建时带有 **`WS_EX_NOREDIRECTIONBITMAP`**（没有 GDI 内容）
> - Shell DefView 子窗口是 `WS_EX_LAYERED` 子窗口

`WS_EX_NOREDIRECTIONBITMAP` 意味着 DWM（Desktop Window Manager）不为该窗口分配重定向表面，而 OpenGL swap chain 依赖 DWM 的重定向表面来合成窗口内容。因此 OpenGL 渲染虽然在 GPU 上正常执行，但输出**无法被 DWM 拾取显示**。

### 1.2 方案选择

| 方案 | 结论 |
|------|------|
| 调整 WorkerW 窗口层级 | 已尝试，无效。问题在渲染层而非窗口管理层 |
| 使用 `WS_EX_LAYERED` 子窗口 + GDI Blt | 可行但性能差（需 CPU readback） |
| **D3D11 替代 OpenGL** | ✅ 采用。D3D11 swap chain 支持 `DXGI_ALPHA_MODE_PREMULTIPLIED`，完全兼容新架构 |
| Vulkan | 风险高：WorkerW 嵌入 + Vulkan swapchain 兼容性未验证；GLSL→SPIR-V 工作量大 |

参考项目情况：
- **Wallpaper Engine**：DirectX 渲染，不受影响
- **Lively**：底层播放器（mpv/VLC/WebView2）使用 DirectX，其 v2.2.0.0 修复的是窗口查找逻辑，而非渲染黑屏

### 1.3 设计原则

- **窗口模式保留 OpenGL 不动**——稳定运行，无需迁移
- **壁纸模式使用 D3D11**——仅在 `--wallpaper` 时激活
- **GLSL shader 文件格式不变**——运行时翻译为 HLSL，用户无感知
- **模块接口与 OpenGL 版本一一对齐**——便于维护和对比
- **所有 D3D11 代码包裹在 `#ifdef _WIN32` 中**——不影响其他平台

## 2. 架构概览

### 2.1 双渲染后端

```
main.cpp 入口
  ├─ config.wallpaperMode = true ?
  │     │
  │     ├─ [D3D11 路径] ──────────────────────────┐
  │     │   不创建 SDL_WINDOW_OPENGL               │
  │     │   SDL 窗口 → 获取 HWND → D3D11 SwapChain │
  │     │   useD3D11 = true                        │
  │     │                                          │
  │     └─ [OpenGL 路径] (已停用，代码保留)          │
  │                                                │
  └─ config.wallpaperMode = false ?                │
        │                                          │
        └─ [OpenGL 路径] ── SDL_WINDOW_OPENGL       │
            GL context + GLAD                      │
                                                   │
渲染循环                                            │
  ├─ if (useD3D11)  ← D3D11 壁纸渲染路径 ──────────┘
  │     D3D11MultiPass → D3D11Renderer → Present
  │
  └─ else           ← OpenGL 渲染路径
        MultiPassRenderer → SDL_GL_SwapWindow
```

### 2.2 模块对照

| 功能 | OpenGL 模块 | D3D11 模块 | 说明 |
|------|------------|-----------|------|
| 设备/上下文 | SDL_GL + GLAD | `D3D11Renderer` | Device/Context/多 SwapChain |
| Shader 编译 | `ShaderManager` | `D3D11ShaderManager` | HLSL 模板 + D3DCompile |
| Shader 翻译 | — | `glsl_to_hlsl` | GLSL → HLSL 文本翻译 |
| 多 Pass 渲染 | `MultiPassRenderer` | `D3D11MultiPass` | Buffer/Image/CubeMap pass |
| 纹理管理 | `TextureManager` | `D3D11TextureManager` | stb_image + SRV |
| 降分辨率 | `BlitRenderer` | `D3D11BlitRenderer` | 低分辨率 RTV + blit |
| 全屏几何 | `Renderer` (VAO/VBO) | `D3D11Renderer::GetFullscreenVS()` | 全屏三角形 VS |

### 2.3 依赖关系

```
D3D11Renderer (设备层)
  ├─ ID3D11Device / ID3D11DeviceContext
  ├─ IDXGISwapChain1 × N（每个显示器）
  └─ 全屏三角形 VertexShader（所有模块共享）

D3D11MultiPass (渲染编排层)
  ├─ 持有 D3D11ShaderManager × M (每个 pass 一个)
  │     └─ 调用 TranslateGlslToHlsl() + D3DCompile()
  ├─ RTV/SRV 双缓冲纹理链
  └─ 依赖 D3D11Renderer 的 Device/Context

D3D11TextureManager (资源层)
  ├─ stb_image 加载 → ID3D11Texture2D + SRV
  └─ 为 D3D11MultiPass 提供外部纹理

D3D11BlitRenderer (后处理层)
  ├─ 降分辨率 RTV + blit PS
  └─ 每个显示器独立实例
```

## 4. 关键实现细节

### 4.1 GLSL → HLSL 翻译 (`glsl_to_hlsl.cpp`)

翻译器接口设计为**可替换**——`TranslateGlslToHlsl(const std::string&) -> std::string` 是唯一入口，当前基于正则/文本替换，后续可无缝切换到 SPIRV-Cross。

#### 翻译流水线

```
输入: ShaderToy 用户 GLSL 代码（不含 uniform 声明）
  │
  ├── Step 1: 预处理
  │     移除 #version 指令、precision 声明
  │
  ├── Step 2: 类型替换（全词匹配）
  │     vec2/3/4   → float2/3/4
  │     ivec2/3/4  → int2/3/4
  │     bvec2/3/4  → bool2/3/4
  │     uvec2/3/4  → uint2/3/4
  │     mat2/3/4   → float2x2/3x3/4x4
  │
  ├── Step 3: 内置函数替换
  │     mix        → lerp
  │     fract      → frac
  │     inversesqrt→ rsqrt
  │     dFdx/dFdy  → ddx/ddy
  │     mod        → glsl_mod（注入辅助函数）
  │
  ├── Step 3.5: 纹理采样重写
  │     texture(iChannel0, uv)         → texture_iChannel0(uv)
  │     textureLod(iChannel0, uv, lod) → textureLod_iChannel0(uv, lod)
  │     texelFetch(iChannel0, c, lod)  → texelFetch_iChannel0(c, lod)
  │     （这些宏在 HLSL 模板中定义，绑定对应的 sampler）
  │
  └── Step 4: 注入辅助函数
        glsl_mod() — 多重载版本，保持 GLSL mod 语义
        （结果与 y 同号，不同于 HLSL fmod 与 x 同号）

输出: HLSL 代码段（仍需 D3D11ShaderManager 包装 cbuffer + 纹理声明 + main）
```

#### 已知局限

| 问题 | 影响 | 当前处理 | 后续方案 |
|------|------|---------|---------|
| 矩阵乘法顺序 | GLSL 列优先 `m*v`，HLSL 行优先 `mul(v,m)` | 大部分 ShaderToy shader 不显式使用矩阵乘法 | SPIRV-Cross |
| `atan(y,x)` 双参数 | HLSL 需要 `atan2(y,x)` | 未全局替换（避免误改单参数 `atan`） | SPIRV-Cross |
| 复杂 swizzle 嵌套 | 如 `v.yzx.xy` | HLSL 支持，通常无需特殊处理 | — |
| 结构体/数组 | ShaderToy 用户 shader 极少使用 | 直接透传（HLSL 语法兼容） | SPIRV-Cross |

#### SPIRV-Cross 升级路径

```
GLSL 330 ──→ glslang 编译 ──→ SPIR-V ──→ spirv-cross 反编译 ──→ HLSL SM 5.0
```

- 依赖：`glslang`（Khronos 官方 GLSL 编译器）+ `SPIRV-Cross`（Khronos SPIR-V 反编译器），通过 FetchContent 引入
- 优点：语义级翻译，100% 正确处理所有 GLSL 语法和语义

### 4.2 HLSL 模板 (`D3D11ShaderManager::WrapShaderToyHlsl`)

`D3D11ShaderManager` 将翻译后的用户代码包装成完整的 HLSL Pixel Shader：

```hlsl
// 1. Constant Buffer（与 C++ ShaderToyConstants 结构体字节对齐）
cbuffer ShaderToyUniforms : register(b0) {
    float4 iResolution;     // xyz=分辨率, w=padding
    float  iTime;
    float  iTimeDelta;
    int    iFrame;
    float  iFrameRate;
    float4 iMouse;
    float4 iDate;
    float  iSampleRate;
    float  iClickTime;
    float2 _pad0;           // 16 字节对齐 padding
    float4 iChannelTime;
    float4 iChannelResolution[4];
    float4 _cubeFaceRight;  // CubeMap pass 用
    float4 _cubeFaceUp;
    float4 _cubeFaceDir;
};

// 2. 纹理 + 采样器声明（根据 channelTypes_ 动态生成）
Texture2D<float4> iChannel0 : register(t0);  // 或 TextureCube / Texture3D
SamplerState sampler0 : register(s0);
...

// 3. texture 兼容函数
float4 texture(Texture2D<float4> tex, SamplerState samp, float2 uv) { ... }
float4 textureLod(...) { ... }
float4 texelFetch(...) { ... }

// 4. texture_iChannelN 宏（将翻译器重写的调用绑定到正确的 sampler）
#define texture_iChannel0(uv) texture(iChannel0, sampler0, (uv))
#define textureLod_iChannel0(uv, lod) textureLod(iChannel0, sampler0, (uv), (lod))
...

// 5. Common 共享代码（如果有）
// ... 用户 common.glsl 翻译后的 HLSL ...

// 6. 用户翻译后的 HLSL 代码（含 mainImage）
// ... TranslateGlslToHlsl() 的输出 ...

// 7. PS 入口点
float4 main(float4 pos : SV_Position) : SV_Target {
    float2 fragCoord = float2(pos.x, iResolution.y - pos.y);  // Y 翻转
    float4 fragColor = float4(0, 0, 0, 1);
    mainImage(fragColor, fragCoord);
    return fragColor;
}
```


### 4.4 全屏三角形 VS

使用一个超大三角形覆盖全屏（比全屏四边形少 1 个顶点，减少 rasterizer 工作）：

```hlsl
// 无需顶点缓冲，SV_VertexID 生成坐标
float4 main(uint id : SV_VertexID) : SV_Position {
    float2 uv = float2((id << 1) & 2, id & 2);
    return float4(uv * float2(2, -2) + float2(-1, 1), 0, 1);
}
```

所有 pass（Buffer/Image/CubeMap/Blit）共享此 VS，由 `D3D11Renderer` 创建并管理。

### 4.5 多 Pass 渲染 (`D3D11MultiPass`)

#### Pass 类型

| Pass 类型 | RTV | 分辨率 | 双缓冲 |
|-----------|-----|--------|--------|
| Buffer A/B/C/D | `ID3D11Texture2D` (RGBA16F) | config.width × config.height | ✅ 交换 front/back |
| Image | SwapChain back buffer 或降分辨率 RTV | 显示器分辨率或缩放后 | ❌ |
| CubeMap | `ID3D11Texture2D` (Cube, 6 面 RTV) | 1024 × 1024 | ✅ 交换 front/back |

#### 渲染流程（壁纸模式）

```
每帧:
  1. RenderBufferPasses()  ← 所有显示器共享，只渲染一次
     ├─ CubeMap pass（如果有）× 6 面
     ├─ Buffer A pass → 输出到 bufferA.outputRTV
     ├─ Buffer B pass → 输出到 bufferB.outputRTV
     ├─ ...
     └─ SwapBuffers()   ← 交换所有 pass 的 front/back 纹理

  2. 遍历显示器:
     ├─ 跳过被遮挡的显示器
     ├─ 鼠标坐标转局部
     ├─ RenderImagePass()
     │   ├─ [降分辨率] 渲染到 d3dBlit.RTV → BlitToTarget(swapChain.RTV)
     │   └─ [全分辨率] 直接渲染到 swapChain.RTV
     └─ Present()
```

#### 纹理输入绑定

每个 pass 有 4 个输入通道（iChannel0~3），通过 `inputChannels[]` 映射：

| inputChannels[i] 值 | 含义 | 绑定的 SRV |
|--------------------|------|-----------|
| 0~3 | Buffer pass 索引 | `bufferPasses_[idx].outputSRVPrev`（上一帧输出） |
| 100+i | 外部纹理通道 i | `externalTextures_[i].srv` |
| 200 | CubeMap pass 输出 | `cubeMapPass_.cubeMapSRVPrev` |
| -1 | 未绑定 | 不绑定 |

### 4.6 SwapChain 管理 (`D3D11Renderer`)

壁纸模式支持多显示器，每个显示器一个独立的 SwapChain：

```
D3D11Renderer
  ├─ device_     (共享)
  ├─ context_    (共享)
  └─ swapChains_[]
       ├─ [0] Monitor 0: HWND → IDXGISwapChain1 → RTV
       ├─ [1] Monitor 1: HWND → IDXGISwapChain1 → RTV
       └─ ...
```

SwapChain 创建参数：
- `DXGI_SWAP_EFFECT_FLIP_SEQUENTIAL`
- `DXGI_FORMAT_R8G8B8A8_UNORM`（back buffer）
- `BufferCount = 2`（双缓冲）
- `SyncInterval = 0`（VSync 关闭，由 SDL_Delay 控帧）

### 4.7 降分辨率渲染 (`D3D11BlitRenderer`)

`renderScale < 1.0` 时启用（壁纸模式默认 0.5x），每个显示器独立实例：

```
Shader 渲染 → 降分辨率 RTV (rW × rH)
                    │
              Blit PS（线性插值放大）
                    │
              SwapChain back buffer (fullW × fullH)
```
